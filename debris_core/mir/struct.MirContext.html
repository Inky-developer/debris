pub struct MirContext<'ctx> {
    pub compile_context: &'ctx CompileContext,
    pub span: Span,
    pub kind: ContextKind,
    pub id: ContextId,
    pub nodes: Vec<MirNode>,
    pub control_flow: ControlFlowMode,
    pub return_values: ReturnValues,
    jump_location_counter: usize,
}
Expand description

Keeps track of single context, which can be a function, a loops or something else. As a rule of thumb, everything which has its own namespace is a context.

Fields

compile_context: &'ctx CompileContext

A ref to the global compile context

span: Spankind: ContextKindid: ContextId

The context id and the id of the corresponding namespace

nodes: Vec<MirNode>

All mir nodes that are emitted

control_flow: ControlFlowMode

Which control flow mode is used for this context Most context just do nothing on end, but it is also possible to return from a function or to break from a loop

return_values: ReturnValues

The values returned by this function

jump_location_counter: usize

Holds the current jump location index
