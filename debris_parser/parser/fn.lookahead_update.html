fn lookahead_update(parser: &mut Parser<'_>) -> bool
Expand description

Looks forward and checks if the current item can be an update. This is a hack to resolve the statement-ambiguity where both of expr and update are valid. For example, (a, b, c) could be parsed as a tuple or as a pattern for an update.

IMPORTANT: This function does not verify that the update is valid, it just verifies that there is something pattern-like followed by an assign op.
